Costo de busqueda en arbol: 
    Se recorre UNA rama del arbol completa (peor caso).
    Suponiendo que n es la cantidad de elementos de un arbol, que cantidad de elementos tiene una rama?
        n = cant de elementos arbol
        h = cant de elementos rama 
            n = 2^h-1
    Suponiendo esto: n = 2^h-1, cuanto vale h?
    El numero h tal que 2^h es igual a n.
        Se llama "logaritmo en base 2" 
    h = log n 
        h = cuantas veces hay que multiplicar por 2 para dar n?

Costo logaritmico : Esta entre constante y lineal. O(log n)
Eneloguene : Hasta operaciones logaritmicas por cada elemento O(n log n)

El costo logaritmico en arbol se mantiene si este esta balanceado, de ahi sale el logaritmo, pero lineal 
sigue siendo el peor caso.

Costo Eneloguene: n tamaño maximo de un arbol, t cantidad de arboles. 
Cada operacion es logaritmica en un arbol: O(log n)
Hay t operaciones: el costo es O(t log n)

Para mejorar eficiencia de TADs anteriores, necesitamos invariantes, para garantizar condiciones.
Binary Search Tree (BST): data Tree a = EmptyT | NodeT a (Tree a) (Tree a)
Invariante de BST: (NodeT a ti ta)
                    Todos los elementos de ti son menores que a 
                    Todos los elementos de td son mayores que x 
                    ti y td tambien cumplen invariante BST.
En el peor caso, no esta balanceado, costo lineal, si esta balanceado, logaritmico.
Cuando borro en un BST, queda desbalanceado, debo agarrar un elemento para que siga siendo BST, dos posibles
candidatos son: del subarbol izquierda, el que este mas a la derecha, y el del subarbol derecho, el que este 
mas a la izquierda, debo rearmar. 

Arboles balanceados: AVLs 
    Para mantener balance, necesito invariantes.
    Invariante de AVL: en NodeT x ti td 
        La diferencia de alturas entre ti y tf es menor o igual a 1 
        ti y d tambien cumplen el invariante de AVL 

    Rebalanceo de AVLs, un elemento debe subir, y otro bajar. 
    Cuando tengo diferencia de 2, sube el del medio y bajan los dos, el menor del del medio va a la izquierda
    el mayor a la derecha, asi sigo garantizando BST 
    Para implementarlo eficiente, debe mantenerse la altura:
        data AVL a = EmptyAVL | NodeAVL Int a (AVL a) (AVL a)

    Como garantizo el invariante de AVL?
        Al insertar y borrar, se requiere ver cuales lugares pueden alterar eso.
            Donde se reconstruyen AVLs 
            Cada nodo debe revisarse

Costos logaritmicos en PQ.
    Hay que usar arboles, llamados heaps 
    Invariante Heap:
        La raiz es el minimo de todos los elementos
        Los subarboles cumplen el invariante de heap 
    Invariante full tree:
        Todos los niveles salvo, quizas el ultimo, estan completos 
        El ultimo no tiene agujeros de izquierda a derecha
    Ventajas de full tree: 
        Tiene un unico lugar donde insertar y donde borrar 
        Al usar memoria, se puede implementar muy eficiente
    Desventajas en el modelo declarativo:
        Se precisan datos adicionales para ubicar el lugar de insercion/borrado
    Al combinar ambos invariantes, se puede deducir las operaciones de insercion y borrado

data Dir = Izq | Der 
data PriorityQueue a = PQ [Dir] (Tree a)
    INV.REP: en (PQ pos t)
        t cumple ser un heap
        t cumple ser un arbol lleno 
        pos indica el camino hasta la posicion de insercion en t
